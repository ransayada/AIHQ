1. PROJECT OVERVIEW & VISION
==============================================================================

AIHQ is a SaaS Digital Audio Workstation that runs entirely in the browser.
It combines professional-grade DAW features (step sequencer, piano roll,
mixer, effects rack) with AI-powered tools (Claude AI assistant, Magenta.js
pattern generation, AI mastering suggestions).

Target users: electronic music producers, beatmakers, sound designers.
Business model: Freemium (Free / Pro / Studio plans) with Stripe billing.

Live URL: Deployed on Vercel (see README for URL)
API URL: Deployed on Render/Fly.io

2. TECHNOLOGY STACK
==============================================================================

Frontend:
  - Next.js 15.5 (App Router, React 19, "use client" components)
  - TypeScript 5.x (strict mode)
  - Tailwind CSS v4 (utility-first, custom DAW color tokens)
  - Zustand 5.x (state management — one store per domain)
  - Radix UI (accessible primitives: Tabs, DropdownMenu, Dialog, Tooltip)
  - Lucide React (icons)
  - Tone.js 15.x (via @aihq/audio-engine — NEVER import directly in web)
  - Light/Dark mode via data-theme attribute on <html> + CSS var overrides

Backend:
  - Hono v4 (ultra-fast web framework, runs on Node.js)
  - @hono/node-server (production HTTP adapter)
  - Prisma ORM (PostgreSQL)
  - Clerk (authentication — currently bypassed in dev with mock passthrough)
  - Stripe (billing — via @stripe/stripe-js)
  - Winston (structured logging, optional Elasticsearch transport)
  - WebSocket (ws package — Yjs CRDT for real-time collaboration)

AI / Audio:
  - Anthropic Claude API (claude-sonnet-4-6, streaming)
  - Magenta.js (Google — DrumsRNN + MelodyRNN, runs client-side in browser)
  - Tone.js (synthesis, sequencing, effects, transport)
  - Web MIDI API (MIDI controller mapping)
  - MediaRecorder API (audio export)
  - Yjs (CRDT for conflict-free collaborative editing)

Infra / Dev:
  - Turborepo 2.x (monorepo task runner with caching)
  - pnpm 10.x (workspace package manager)
  - Vitest 3.x (unit + component tests)
  - Playwright (E2E tests)
  - Storybook 8.x (UI component stories, in @aihq/ui package)
  - Docker Compose (ELK stack: Elasticsearch + Kibana)

4. GETTING STARTED (LOCAL DEV SETUP)
==============================================================================

Prerequisites:
  - Node.js 20+ (LTS)
  - pnpm 10+ (`npm i -g pnpm`)
  - Docker Desktop (for ELK stack, optional)

Step 1: Clone and install
  git clone <repo-url>
  cd aihq
  pnpm install

Step 2: Configure environment variables

  apps/api/.env:
    NODE_ENV=development
    PORT=3001
    # Optional — leave blank to use in-memory mock DB:
    DATABASE_URL=postgresql://user:pass@localhost:5432/aihq
    # Optional — enables Claude AI chat:
    ANTHROPIC_API_KEY=sk-ant-...
    # Optional — enables Clerk auth (otherwise dev passthrough is used):
    CLERK_SECRET_KEY=sk_test_...
    # Optional — enables Stripe billing:
    STRIPE_SECRET_KEY=sk_test_...
    STRIPE_WEBHOOK_SECRET=whsec_...

  apps/web/.env:
    NEXT_PUBLIC_API_URL=http://localhost:3001
    NEXT_PUBLIC_WS_URL=ws://localhost:3001
    NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...

Step 3: Run everything
  pnpm turbo dev          # All packages in dev mode

  Or individually:
  pnpm --filter @aihq/api dev       # API on :3001
  pnpm --filter @aihq/web dev       # Web on :3000

Step 4: Access the app
  Web UI:      http://localhost:3000
  API:         http://localhost:3001
  Swagger UI:  http://localhost:3001/docs
  OpenAPI JSON:http://localhost:3001/docs/openapi.json

Step 5 (optional): ELK Stack
  docker-compose -f docker-compose.elk.yml up -d
  Kibana: http://localhost:5601
  Elasticsearch: http://localhost:9200

5. ARCHITECTURE DEEP-DIVE
==============================================================================

Request Flow:
  Browser → Next.js (port 3000) → API (port 3001, /api/* proxied via next.config.ts)

Authentication Flow:
  - In LOCAL DEV: auth middleware automatically passes through with a mock
    dev-clerk-user / dev-db-user / STUDIO tier. No Clerk setup needed.
  - In PRODUCTION: Clerk JWT verified in auth.ts middleware. User must exist
    in the database (synced via /api/users/sync Clerk webhook).

Audio Signal Chain:
  MIDI/Keyboard → Instrument (SynthDrumKit | Synthesizer | Sampler)
               → instrument AudioNode
               → EffectsChain (reverb → delay → EQ → compressor → distortion)
               → TrackManager mixerAudioChannel (volume/pan fader)
               → Tone.getDestination() (master output)

Real-Time Collaboration (Yjs):
  Browser A → WebSocket (/collab?projectId=X) → CollabServer
  CollabServer → Y.Doc per project → broadcast Yjs updates to all clients
  Browser B ← receives update → Y.applyUpdate → local store syncs

6. PACKAGE DESCRIPTIONS
==============================================================================

@aihq/shared (packages/shared)
  - Zod schemas for all data types: Track, Clip, MidiNote, SynthPreset,
    StepPattern, ProjectData, etc.
  - PLAN_LIMITS constants (maxProjects, maxAiGenerations per tier)
  - PROJECT_TEMPLATES: 6 genre presets used in CreateModal
  - Compiled to packages/shared/dist, imported by all other packages

@aihq/ui (packages/ui)
  - Design system: Button, Knob, Slider, Panel, Meter, Avatar, DJDeckCard
  - CSS custom properties (DAW color tokens: --color-studio-*, --color-accent-*)
  - Storybook stories for all components
  - Vitest tests in src/__tests__/

@aihq/audio-engine (packages/audio-engine)
  - AudioEngine singleton: getInstance() → single shared engine instance
  - CRITICAL: All Tone.js usage MUST stay in this package.
    apps/web must NEVER import from 'tone' directly.
  - Transport: play, stop, seek, BPM, position tracking, metronome
  - Sequencer: 16-step pattern sequencer, per-track patterns
  - Synthesizer: polyphonic synth (osc types, ADSR, filter)
  - SynthDrumKit: 9 synthesized drum pads (no samples required)
  - EffectsChain: reverb, delay, EQ3, compressor, distortion (all Tone.js)
  - MIDIManager: Web MIDI API wrapper, CC mapping, learn mode
  - StemExporter: MediaRecorder-based audio capture + download
  - MagentaEngine: DrumsRNN + MelodyRNN (loads TF.js models from CDN)

@aihq/api (apps/api)
  - Hono v4 REST API + WebSocket server
  - In dev without DATABASE_URL: uses in-memory Maps as mock DB
  - Routes mounted at /api/* (except /health, /webhooks, /logs, /docs)
  - All /api/* routes protected by authMiddleware (dev passthrough in local)

@aihq/web (apps/web)
  - Next.js 15 App Router with "use client" DAW components
  - Proxies /api/* to localhost:3001 (configured in next.config.ts)

  GET  /api/samples/:id/stream          Stream/download sample
  DELETE /api/samples/:id              Delete sample

  POST /api/ai/chat                     Claude AI chat (streaming text)
  POST /api/ai/log                      Log Magenta generation
  GET  /api/ai/usage                    AI generation usage stats

  GET  /api/users/me                    Current user profile
  POST /api/billing/checkout            Stripe Checkout session
  POST /api/billing/portal              Stripe billing portal

WebSocket: ws://localhost:3001/collab?projectId=X&userId=Y&userName=Z
  Binary protocol:
    0x01 + Yjs update bytes = broadcast document update
    0x02 = sync request (server sends full state)
    0x03 + JSON bytes = presence update (array of {userId, userName, color})

9. AUDIO ENGINE (TONE.JS)
==============================================================================

The AudioEngine is a singleton initialized lazily on first user interaction:
  import { audioEngine } from "@aihq/audio-engine";
  await audioEngine.initialize(); // creates Tone.js AudioContext

CRITICAL RULE: apps/web must NEVER import from 'tone' directly.
All Tone.js code stays in packages/audio-engine.

Key singleton surfaces:
  audioEngine.transport      — play(), stop(), seek(), BPM, onPositionChange()
  audioEngine.sequencer      — setPattern(), setStepActive(), start/stop
  audioEngine.trackManager   — registerTrack(), setVolume(), setPan(), etc.
  audioEngine.magenta        — init(), generateDrums(temp), generateMelody(seed, temp)
  audioEngine.isInitialized  — boolean, check before calling audio methods

Instrument types:
  "drum"  → SynthDrumKit (9 synthesized pads, no samples required)
  "synth" → Synthesizer (polyphonic, ADSR, osc type, filter)
  "bass"  → Synthesizer (mono bass preset)

10. STATE MANAGEMENT (ZUSTAND)
==============================================================================

All stores are in apps/web/src/stores/. Each store follows the pattern:
  export const useXxxStore = create<XxxState>()(devtools(immer((set, get) => ({...}))))

Stores:
  transportStore    — isPlaying, isRecording, bpm, position, metronome
  tracksStore       — tracks[], selectedTrackId, all track mutations
  uiStore           — activeBottomPanel, isAIPanelOpen (BottomPanel union type)
  aiStore           — chatMessages, streamingMessage, magentaLoaded, isGenerating
  djStore           — decks[], crossfader, eq settings
  samplesStore      — samples[], loading, uploading, fetchSamples, uploadSample
  midiStore         — status, deviceNames, mappings, learningId (persisted)
  versionStore      — snapshots[], loading, saving, fetchSnapshots, saveSnapshot
  shareStore        — shareToken, shareUrl, loading, createShare, revokeShare
  pluginStore       — plugins[], togglePlugin (8 built-in plugins, persisted)
  collabStore       — status, users[], doc (Y.Doc), connect(), disconnect()

The hook useAudioSync (apps/web/src/hooks/useAudioSync.ts) bridges Zustand →
audio engine reactively: when tracks change, it calls trackManager APIs.

11. AUTHENTICATION & AUTHORIZATION
==============================================================================

Dev mode (no CLERK_SECRET_KEY set):
  - authMiddleware passes all requests with:
    clerkUserId: "dev-clerk-user"
    dbUserId:    "dev-db-user"
    tier:        "STUDIO"
  - All features are accessible without login

Production mode:
  - Clerk JWT verified via @clerk/backend verifyToken()
  - User must exist in DB (synced via /webhooks/clerk POST — user.created event)
  - Plan tiers: FREE, PRO, STUDIO (from Prisma subscription table)
  - AI chat requires PRO or STUDIO tier
  - Free tier: limited projects, no AI chat, no AI generation

CORS: Only origins in ALLOWED_ORIGINS env var (default: http://localhost:3000)

13. KEY FEATURES & THEIR IMPLEMENTATION
==============================================================================

Step Sequencer
  Component:   StepSequencer.tsx
  Store:       tracksStore (stepPattern per track)
  Audio:       audioEngine.sequencer.setPattern()
  Each drum track has a 16-step boolean array + velocities array.
  StepButton components are memoized to prevent re-renders.

Piano Roll
  Component:   PianoRoll.tsx, NoteGrid.tsx
  Store:       tracksStore (clips[].notes[])
  Canvas-based (not DOM nodes) for 88-key × many-beats performance.
  Supports: step record mode, drag-to-move notes, note resize.

Mixer
  Component:   Mixer.tsx, ChannelStrip.tsx
  Store:       tracksStore (volume, pan, muted, soloed)
  Audio:       audioEngine.trackManager.setVolume/setPan/setMuted
  Live VU meters via Web Audio AnalyserNode → Meter canvas component.

AI Chat (Claude)
  Component:   ChatInterface.tsx → POST /api/ai/chat
  The endpoint streams text (Hono stream()). ChatInterface reads via
  response.body.getReader() and appends tokens to streamingMessage in aiStore.
  Requires ANTHROPIC_API_KEY in apps/api/.env.

AI Mastering
  Component:   MasteringPanel.tsx → POST /api/ai/chat (streaming)
  Asks Claude to return a JSON object with EQ, compression, loudness settings.
  Parses the JSON from the streamed response and displays visual results.

Magenta.js AI Generation
  Component:   AIPanel.tsx (Drums + Melody tabs)
  Engine:      audioEngine.magenta.init() / generateDrums() / generateMelody()
  Loads TF.js models from CDN on first use. Runs entirely client-side.

Real-Time Collaboration (Yjs)
  Store:       collabStore.ts (Yjs Y.Doc + WebSocket)
  Server:      apps/api/src/collab/CollabServer.ts
  Protocol:    Binary frames — 0x01=Yjs update, 0x02=sync, 0x03=presence
  CollabPresence.tsx renders connection status + other users' avatars.

MIDI Controller
  Store:       midiStore.ts (persisted to localStorage)
  Engine:      packages/audio-engine/src/midi/MIDIManager.ts
  Connect → Web MIDI API requestMIDIAccess()
  CC Mapping → Learn mode captures next incoming CC
  Dispatches window CustomEvents: "midi:cc" and "midi:noteon"

Audio Export
  Component:   ExportDialog.tsx
  Engine:      StemExporter (MediaRecorder on AudioContext destination)
  Flow: Record → play project → stop → download WebM/Opus file

Version History (Snapshots)
  Component:   VersionHistoryPanel.tsx
  API:         GET/POST/DELETE /api/projects/:id/snapshots
  Captures: { tracks: Track[], bpm: number } → restore restores both

Project Templates
  Data:        packages/shared/src/templates.ts (6 presets)
  Component:   TemplateSelector.tsx (shown in CreateModal)
  When selected, POST /api/projects with templateId. Server applies
  template tracks and BPM to the new project.

Public Sharing
  Component:   SharePanel (via shareStore) + /share/[token] page
  API:         POST /api/projects/:id/share → returns { token, url }
  Share page: animated waveform, track list, copy link, CTA

Performance Mode
  Component:   PerformanceView.tsx (fixed overlay)
  8 pads mapped to Q/W/E/R/A/S/D/F keys
  Space = Play/Stop | Fullscreen API | MIDI noteon events

Plugin Browser
  Store:       pluginStore.ts (8 built-in virtual plugins, persisted)
  Categories:  dynamics, fx, utility, creative
  Currently cosmetic — plugins activate/deactivate in the store.
  Future: wire to EffectsChain per plugin type.

ELK Stack Logging
  Backend:  apps/api/src/lib/logger.ts (Winston + optional ES transport)
  Frontend: apps/web/src/lib/logger.ts (buffered, ships to POST /logs)
  Start ELK: docker-compose -f docker-compose.elk.yml up -d
  View logs: http://localhost:5601 (Kibana)

17. CI/CD & DEPLOYMENT
==============================================================================

Build pipeline (turbo.json):
  build → test → lint  (in order, respects package dependencies)
  pnpm turbo build    # builds all 5 packages
  pnpm turbo test     # runs all tests after build
  pnpm turbo lint     # ESLint all packages

Deployment:
  Frontend: Vercel (auto-deploys from main branch)
    - NEXT_PUBLIC_API_URL points to production API
    - NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY for auth
  Backend: Render or Fly.io (Docker or Node process)
    - DATABASE_URL, CLERK_SECRET_KEY, ANTHROPIC_API_KEY, STRIPE_SECRET_KEY

Environment files (never commit):
  apps/api/.env
  apps/web/.env

For production, all in-memory mock services must be replaced with real DB:
  - mockProjects → Prisma Project model
  - In-memory snapshots → Prisma Snapshot model (add to schema)
  - In-memory shares → Prisma Share model (add to schema)
  - In-memory samples → S3/R2 bucket + DB record

19. COMMON PITFALLS & GOTCHAS
==============================================================================

1. "Cannot use 'tone' in apps/web"
   → Always import from '@aihq/audio-engine', never 'tone'

2. "audioEngine.initialize() must be called after user gesture"
   → Web Audio context must start from a user interaction (click/keypress)
   → AudioEngine handles this lazily via useAudioEngine hook

3. "Hydration mismatch in Next.js"
   → All DAW components use "use client" — they SSR as empty placeholders
   → Zustand stores initialize client-side; use useEffect for audio init

4. "Hono route type error on c.get('dbUserId')"
   → Use new Hono<{ Variables: AuthVariables }>() not new Hono()

5. "CollabPresence shows error state"
   → WebSocket tries ws://localhost:3001/collab — ensure API server is running
   → In production set NEXT_PUBLIC_WS_URL to wss://your-api-domain.com

6. "AI chat returns error: PLAN_REQUIRED"
   → In local dev, tier is always "STUDIO" — this shouldn't happen
   → If it does, check auth middleware is using the dev passthrough

7. "MediaRecorder export produces silent file"
   → ExportDialog needs AudioContext from audioEngine transport
   → Must click Record AFTER audioEngine is initialized

8. "Magenta.js fails to load"
   → Requires internet connection to download TF.js model files from CDN
   → Check browser console for model download errors

9. "pnpm turbo build fails with cache errors"
   → Run: pnpm turbo build --force  (bypass cache)
   → Or delete .turbo/ directory

10. "TypeScript error: immer draft type"
    → When using immer, mutations on draft state are fine
    → For return values from immer callbacks, return new state explicitly
